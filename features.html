Customizable UI Features and Animation Effects

Font Size Slider

The slider at the top right of the page allows users to adjust font size dynamically. The slider updates the root html font size, so it affects all the text. By setting the minimum and maximum font size, the webpage can be customized.

How it works:

  • By using a slider input < input type="range"> users can adjust font size.
  • Javascript will detect the slider movement and update the font size dynamically by changing the font size of html.
  • The user selection is not stored so page will reset when reloaded.


Dark Mode Toggle

By clicking the dark mode button at the top right of the page, the theme can be switched. The user's preference is stored in localStorage, so dark mode persists across page reloads.

How it works:

  • The :root CSS variables define default light mode colors.
  • A .darkmode class overrides these variables when activated.
  • The button toggles between light and dark modes.
  • The JavaScript script stores the user's preference in localStorage to persist the theme across page reloads.
The colours for dark mode and light mode can be defined and customized.

On-Scroll Animations

A scroll-driven animation is an animation that responds to scrolling. There's a direct link between scrolling progress and the animation's progress. Scroll-driven animations pause, play, and run with the direction of the scroll.

On-scroll animations improve user engagement by revealing content gradually as the user scrolls down. This animation is made using animation-timeline and animation-range properties which are part of the newer Scroll-driven Animations API, which currently has limited browser support (only Chrome).

The view div contains all the animated blocks and each block is a seperate section that will animate when you scroll to it.

How it works:

  • The CSS @keyframes rule defines an animation effect. from and to are used to define how the block with start and end.
  • animation: appear linear tells the script to 'appear' the animation created.
  • The animation-timeline controls the animation's progress. view() triggers the animation when the element enters the viewport.
  • The animation-range property ensures that elements animate smoothly as they scroll into view. Here the animation starts when the block enters the screen and completes at 40%.

Disabling Animations

Disabling Animations can be easily done by setting the animations to none inside a .no-animations class. This overrides the animation properties and stops the animation.
